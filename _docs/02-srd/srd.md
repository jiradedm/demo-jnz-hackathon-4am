# SRD: System Requirement Document

## 1. System Overview

ระบบรับข้อมูลจากผู้ใช้ (ฟอร์มหรือการอัปโหลด) แล้วสร้าง pitch deck เป็นชุด slide แสดงผลบนเว็บในรูปแบบคล้ายการดู deck (หลาย slide / ลำดับชัดเจน) ผู้ใช้แก้ไขเนื้อหาหรือการจัดวางผ่านแชทแบบ realtime และเมื่อพอใจแล้วมีขั้นตอนดาวน์โหลดผลลัพธ์ ภาพรวมและ style สอดคล้องกับ brand Winitch แบบ modern ในรอบนี้การสร้างเนื้อหาและดาวน์โหลดเป็น mock ได้ แต่ flow และปฏิสัมพันธ์ต้องให้ความรู้สึกว่าเป็นระบบที่ทำงานได้จริง

---

## 2. Functional Requirements

- **Input / ข้อมูลเข้า**: ระบบต้องมีฟอร์มหรือช่องทางอัปโหลดสำหรับกรอก/ส่งข้อมูลที่จำเป็นต่อการสร้าง pitch deck (เช่น ชื่อโปรเจกต์ วัตถุประสงค์ จุดเด่น ข้อมูลบริษัท)
- **การสร้าง deck**: ระบบต้องสร้าง slide จากข้อมูลที่ได้รับ และแสดงผลในรูปแบบการดู deck บนเว็บ (มองเห็นหลาย slide และลำดับได้)
- **การแสดงผล slide**: ระบบต้องแสดง slide ที่สร้างขึ้นในรูปแบบคล้าย PowerPoint บนเว็บ รวมถึงการนำทางหรือการมองเห็นหลาย slide ใน deck
- **แชทแก้ไข realtime**: ระบบต้องมีแชทให้ผู้ใช้พิมพ์ prompt เพื่อแก้ไขเนื้อหาหรือการจัดวางของแต่ละ slide และต้องแสดงการอัปเดต (หรือการตอบสนอง) แบบ realtime
- **ดาวน์โหลด**: ระบบต้องมีปุ่มหรือขั้นตอนให้ผู้ใช้ดาวน์โหลด slide เมื่อแก้ไขเสร็จและพอใจแล้ว (ในรอบนี้อนุญาตให้เป็น mock การดาวน์โหลด)
- **Brand และ style**: ระบบต้องใช้ brand identity ตาม Winitch (สี ฟอนต์ ลักษณะภาพรวม) และ style การออกแบบแบบ modern ตามที่กำหนด

---

## 3. Non-Functional Requirements

- **Performance**: การแสดงผล slide และการตอบสนองจากแชทต้องให้ความรู้สึก realtime ไม่ล่าช้าจนกระทบ flow การใช้งาน
- **Security**: ไม่มีข้อกำหนดพิเศษใน scope ปัจจุบัน (ไม่มีระบบสมาชิก/ล็อกอิน)
- **Constraints**: ใช้ mock สำหรับการ generate เนื้อหาจริงและสำหรับการ export เป็นไฟล์ (PowerPoint/PDF) ไม่รองรับหลาย brand หรือ theme นอก Winitch และไม่รวมระบบสมาชิกหรือประวัติการสร้าง deck

---

## 4. High-Level Components

- **Frontend**: ส่วนติดต่อผู้ใช้ ประกอบด้วย ฟอร์ม/ช่องทาง input สำหรับข้อมูล pitch deck, หน้าหรือ view สำหรับแสดง slide แบบ deck, UI แชทสำหรับส่ง prompt แก้ไข, ปุ่ม/ขั้นตอนดาวน์โหลด และการจัด style ตาม Winitch
- **Backend**: ส่วนประมวลผลที่รับข้อมูลจากฟอร์ม/อัปโหลด สร้างหรืออัปเดต slide (ในรอบนี้ใช้ mock), ประมวลคำสั่งจากแชทและส่งผลกลับแบบ realtime และจัดการขั้นตอนดาวน์โหลด (mock ได้)
- **External services**: ไม่กำหนดใน scope ปัจจุบัน (การเชื่อม AI จริงอยู่นอก scope)

---

## 5. Data Overview

- **Input สำหรับสร้าง deck**: ข้อมูลที่ผู้ใช้กรอกหรืออัปโหลด เช่น ชื่อโปรเจกต์ วัตถุประสงค์ จุดเด่น ข้อมูลบริษัท — ใช้เป็นต้นทางสำหรับสร้าง slide
- **Deck / Slide**: โครงสร้างและเนื้อหาของ deck แต่ละชุด ประกอบด้วย slide หลายหน้า แต่ละ slide มีเนื้อหาและการจัดวางที่แก้ไขได้ผ่านแชท
- **แชท / Prompt**: ข้อความที่ผู้ใช้ส่งในแชทเพื่อแก้ไข slide (เช่น ระบุ slide ที่ต้องการและคำสั่ง) และผลลัพธ์การอัปเดตที่ส่งกลับแบบ realtime
- **ผลลัพธ์สำหรับดาวน์โหลด**: สถานะหรือข้อมูลที่ระบบเตรียมสำหรับขั้นตอนดาวน์โหลด (ในรอบนี้เป็น mock ได้ ไม่จำเป็นต้องเป็นไฟล์จริง)
