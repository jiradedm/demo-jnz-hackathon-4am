# BRD: Business Requirement Document
## 1. Objective
ปัญหาที่ต้องการแก้คืออะไร
ทำไมต้องมีระบบ / feature นี้

## 2. Stakeholders
- ผู้ใช้งาน
- ทีมที่เกี่ยวข้อง

## 3. Scope
### In Scope
- สิ่งที่ต้องทำในรอบนี้
### Out of Scope
- สิ่งที่ยังไม่ทำ

## 4. User Scenarios
อธิบายสถานการณ์ใช้งานจริง (เป็นเรื่องเล่าได้)

## 5. Success Criteria
ถือว่าสำเร็จเมื่ออะไรเกิดขึ้น
